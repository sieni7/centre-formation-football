# Contributing to Centre de Formation de Football

## Commit Convention

We use **Conventional Commits**. Please follow this format:

`type(scope): message`

### Types:
- `feat`: A new feature
- `fix`: A bug fix
- `docs`: Documentation changes
- `style`: Changes that do not affect the meaning of the code (white-space, formatting, etc)
- `refactor`: A code change that neither fixes a bug nor adds a feature
- `perf`: A code change that improves performance
- `test`: Adding missing tests or correcting existing tests
- `chore`: Changes to the build process or auxiliary tools and libraries

### Scopes:
- `app`: Route changes
- `ui`: Component/Styling changes
- `domain`: Business logic
- `db`: Supabase/Schema changes
- `infra`: CI/CD, setup

## Development Workflow
1. Create a branch: `feat/feature-name` or `fix/bug-name`
2. Make changes and commit using the convention.
3. Submit a Pull Request.
